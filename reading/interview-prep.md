# Interview Prep — Reading & Watch List

## Coding Practice

- [ ] [Kata-Log: Experienced Katas](https://kata-log.rocks/experienced) — curated coding exercises (TDD, refactoring, SOLID, design)
- [ ] [HackerRank](https://www.hackerrank.com/) — timed problem solving to build interview pacing
- [ ] [LeetCode](https://leetcode.com/) — algorithm and data structure problems (industry standard)
- [ ] [NeetCode](https://neetcode.io/) — curated LeetCode roadmap with video explanations

## Live Coding Interviews

- [ ] [5 Tips for Your Live Coding Interview](https://medium.com/revolut/5-tips-for-your-live-coding-interview-c6d5b05adf97) — practical advice on communicating and working through problems live

## System Design

- [ ] [Designing Data-Intensive Applications](https://www.oreilly.com/library/view/designing-data-intensive-applications/9781491903063/) — Martin Kleppmann's gold standard on distributed systems
- [ ] [System Design Interview, Vol 1](https://www.amazon.com/System-Design-Interview-insiders-Second/dp/B08CMF2CQF) — Alex Xu's practical guide with real interview questions
- [ ] [System Design Interview, Vol 2](https://www.amazon.com/System-Design-Interview-Insiders-Guide/dp/1736049119) — deeper dives into proximity service, stock exchange, etc.
- [ ] [Software Engineering at Google](https://www.oreilly.com/library/view/software-engineering-at/9781492082781/) — lessons on sustainability, scale, and trade-offs
- [ ] [Fundamentals of Software Architecture](https://www.oreilly.com/library/view/fundamentals-of-software/9781492043447/) — architecture patterns and decision-making
- [ ] [Catalog of Patterns of Distributed Systems — Fowler](https://martinfowler.com/articles/patterns-of-distributed-systems/) — consensus, replication, partitioning

## DDD & Architecture Concepts

- [ ] [CQRS — Fowler](https://martinfowler.com/bliki/CQRS.html) — Command Query Responsibility Segregation
- [ ] [DDD Aggregate — Fowler](https://martinfowler.com/bliki/DDD_Aggregate.html) — transaction boundaries, consistency rules
- [ ] [Anemic Domain Model — Fowler](https://martinfowler.com/bliki/AnemicDomainModel.html) — anti-pattern: logic-free domain objects
- [ ] [Ubiquitous Language — Fowler](https://martinfowler.com/bliki/UbiquitousLanguage.html) — shared vocabulary
- [ ] [Microservices — Fowler](https://martinfowler.com/articles/microservices.html) — the canonical article

## Practical Patterns

- [ ] [Feature Toggles — Fowler](https://martinfowler.com/articles/feature-toggles.html) — release management, trunk-based development
- [ ] [CircuitBreaker — Fowler](https://martinfowler.com/bliki/CircuitBreaker.html) — resilience pattern for distributed systems
- [ ] [Dependency Injection — Fowler](https://martinfowler.com/articles/injection.html) — IoC containers
- [ ] [Evolutionary Database Design — Fowler](https://martinfowler.com/articles/evodb.html) — schema migrations strategy
- [ ] [Serverless Architectures — Fowler](https://martinfowler.com/articles/serverless.html) — trade-offs, cold starts, when to use

## AI & GenAI

- [ ] [Emerging Patterns in Building GenAI Products — Fowler](https://martinfowler.com/articles/gen-ai-patterns/) — patterns for LLM-based product development

## Behavioral Interview

- [ ] [Tech Interview Handbook — Behavioral](https://www.techinterviewhandbook.org/behavioral-interview/) — STAR method, common questions
- [ ] Prepare 5-7 stories using STAR format (Situation, Task, Action, Result)
